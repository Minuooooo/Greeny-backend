## Backend Convention
### Issues
✓ `Ex. Title`

- 기능 개발, 오류 수정 등의 Issue 생성
- Issue number 확인 (#1, 2, 3..)
- 트러블 슈팅을 위해 오류에 대한 Issue를 생성하고, 해결 과정 문서화

<br></br>

### Branch
✓ `Ex. Type/#Issue.number-Title`

- `master` : 최종본
- `env` : 환경 설정
- `feat` : 이슈 별 기능 개발
- `refactor` : 리팩토링
- `fix` : 오류 수정
- `test` : 테스트 코드 작성

<br></br>

### Commit Message
✓ `Ex. [ Type ] Title`

- `env` : 환경 설정
- `feat` : 기능 개발
- `refactor` : 리팩토링
- `fix` : 오류 수정
- `chore` : Gradle 설정, 기타 작업
- `test` : 테스트 코드 작성

<br></br>

### PR
✓ `Ex. [#Issue.number] Type: Title`
